Perfect for: sending emails, Slack notifications, anything that shouldn't block the HTTP response. - Internal-call proxy limitation: SAME as @Transactional. Read the JWT from localStorage, add it to the Authorization header. This way you never manually add auth headers — every API call is automatically authenticated. - localStorage vs httpOnly cookie for JWT: localStorage is accessible by JavaScript (XSS risk). httpOnly cookies are not accessible by JS (safer). For a learning project, localStorage is acceptable. In production, httpOnly cookies + CSRF tokens is the more secure pattern. Know the tradeoff. - CORS: browsers block cross-origin requests by default. Your React app on localhost:5173 calling your Spring Boot app on localhost:8080 is cross-origin. Fix: add @CrossOrigin on your controllers, OR configure a global CORS policy in Spring Security (better).
